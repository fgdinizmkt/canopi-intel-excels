# Regra operacional — autorização explícita

Status: ativa no projeto Canopi.

Esta regra vale para a condução do projeto, documentação operacional, auditorias, implementações e sincronizações.

## Regra

Nenhuma etapa, recorte, fase, implementação, fechamento documental, commit, push, atualização local/remota ou mudança de escopo deve acontecer por inferência.

Toda transição precisa de autorização explícita do Fábio.

## Aplicação

- Auditoria não autoriza implementação.
- Validação visual ou técnica não autoriza commit.
- Commit local não autoriza push.
- Fechamento de recorte não autoriza abertura do próximo.
- Recomendação de próximo caminho não é ordem de execução.
- Qualquer agente executor deve receber esta regra nos próximos prompts.
