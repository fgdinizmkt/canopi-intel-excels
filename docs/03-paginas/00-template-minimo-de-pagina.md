# Template mínimo de página

## Objetivo deste documento
Padronizar a estrutura mínima de documentação de qualquer página da Canopi.

## Template oficial

### 1. Papel da página
- objetivo principal
- tipo de página
- para quem ela existe

### 2. Pergunta que a página responde
- qual pergunta principal o usuário resolve aqui
- quais perguntas secundárias são aceitáveis

### 3. O que entra
- inputs de dados
- entidades relacionadas
- sinais consumidos
- dependências de integração

### 4. O que sai
- outputs principais
- decisões geradas
- ações derivadas
- encaminhamentos para outras páginas

### 5. Fronteira da página
- o que esta página faz
- o que ela não deve fazer
- páginas vizinhas com maior risco de sobreposição

### 6. Entidades mais importantes
- entidade principal
- entidades secundárias
- relação com Conta, Contato, Sinal e Ação

### 7. Elementos mínimos de interface
- blocos obrigatórios
- filtros mínimos
- agrupamentos mínimos
- estados mínimos de vazio, erro e destaque

### 8. KPIs ou sinais mínimos
- quais indicadores realmente precisam existir
- quais sinais precisam ser destacados
- o que é apenas opcional ou aprofundamento futuro

### 9. Relação com ABM e ABX
- como a página apoia ABM
- como a página apoia ABX
- se a relação é direta, indireta ou de apoio transversal

### 10. Critério de qualidade
- como saber se a página está boa o suficiente
- o que caracterizaria excesso de escopo
- o que indicaria superficialidade excessiva

## Regra de uso
Nenhuma nova página deve ser considerada pronta sem estar minimamente descrita nesse formato.
