# Perfis de usuário

## Objetivo deste documento
Definir os principais perfis de usuário da Canopi, suas necessidades, nível de profundidade desejado, permissões recomendadas e relação com a plataforma.

## Princípio geral
A Canopi não deve ser desenhada para um único tipo de usuário.

Ela precisa servir diferentes perfis com expectativas distintas de leitura, decisão, execução e governança.

Por isso, o produto deve considerar:
- quem usa
- para quê usa
- com qual frequência usa
- com qual profundidade usa
- com qual permissão usa

## Camadas de usuário
Para fins de produto, os usuários da Canopi podem ser organizados em quatro camadas:
1. núcleo operacional
2. núcleo decisor
3. usuários estendidos
4. usuários ocasionais ou técnicos

## 1. Núcleo operacional
São os perfis que usam a plataforma para operar de forma recorrente.

### Demand Gen / ABM
Usa a Canopi para:
- planejar contas prioritárias
- entender contexto de conta
- acionar campanhas e canais
- acompanhar sinais e ações
- conectar estratégia e execução

Páginas mais relevantes:
- Contas
- Sinais
- Ações
- Estratégia ABM
- Desempenho
- Mídia Paga
- Performance Orgânica

Profundidade esperada:
- alta

Permissão típica:
- edição

### SDR / BDR / Outbound
Usa a Canopi para:
- priorizar abordagem
- entender contatos e contas
- identificar sinais de abertura
- acompanhar cadências e follow-ups
- executar ações relacionadas à prospecção

Páginas mais relevantes:
- Contatos
- Contas
- Sinais
- Ações
- Outbound

Profundidade esperada:
- alta

Permissão típica:
- edição

### Revenue Operations / Marketing Ops
Usa a Canopi para:
- garantir coerência operacional
- acompanhar regras, integrações e dados
- sustentar a lógica do sistema
- analisar gargalos de processo e cobertura

Páginas mais relevantes:
- Visão Geral
- Desempenho
- Integrações
- Configurações
- Sinais
- Ações

Profundidade esperada:
- alta

Permissão típica:
- admin ou edição avançada

## 2. Núcleo decisor
São perfis que usam a Canopi para decidir, acompanhar e orientar.

### Liderança de marketing e receita
Usa a Canopi para:
- acompanhar leitura executiva
- entender prioridades
- monitorar desempenho
- enxergar risco, oportunidade e cobertura
- avaliar se o sistema está gerando consequência operacional

Páginas mais relevantes:
- Visão Geral
- Desempenho
- Sinais
- Contas

Profundidade esperada:
- média

Permissão típica:
- leitura avançada

### Gestão comercial / liderança de contas
Usa a Canopi para:
- acompanhar contas estratégicas
- entender relacionamento e risco
- orientar priorização por conta
- acompanhar ações críticas

Páginas mais relevantes:
- Contas
- Contatos
- Ações
- Sinais
- Orquestração ABX

Profundidade esperada:
- média a alta

Permissão típica:
- leitura avançada ou edição limitada

## 3. Usuários estendidos
São perfis que usam a plataforma em parte do fluxo, não no sistema inteiro.

### Conteúdo / Product Marketing
Usa a Canopi para:
- entender contexto de contas e segmentos
- identificar padrões úteis para mensagem e narrativa
- transformar sinais em temas e materiais

Páginas mais relevantes:
- Contas
- Inteligência Cruzada
- Performance Orgânica
- Assistente

Profundidade esperada:
- média

Permissão típica:
- leitura ou edição limitada

### Mídia Paga
Usa a Canopi para:
- entender impacto por conta, segmento ou campanha
- conectar investimento e contexto operacional
- apoiar priorização de mídia

Páginas mais relevantes:
- Mídia Paga
- Desempenho
- Contas
- Ações

Profundidade esperada:
- média a alta

Permissão típica:
- edição limitada

### CS / AM / Farmer
Usa a Canopi para:
- acompanhar continuidade e expansão
- entender saúde da conta
- orientar relacionamento e proteção de receita

Páginas mais relevantes:
- Orquestração ABX
- Contas
- Contatos
- Sinais
- Ações

Profundidade esperada:
- média a alta

Permissão típica:
- edição limitada

## 4. Usuários ocasionais ou técnicos
São perfis que usam a plataforma com menor frequência ou em situações específicas.

### Diretoria / C-level
Usa a Canopi para:
- leitura consolidada
- visibilidade de risco e prioridade
- acompanhamento de impacto operacional

Páginas mais relevantes:
- Visão Geral
- Desempenho
- Contas

Profundidade esperada:
- baixa a média

Permissão típica:
- leitura

### BI / Dados / Analytics
Usa a Canopi para:
- entender fontes, cobertura e lógica de dados
- validar coerência de leitura
- apoiar evolução do sistema

Páginas mais relevantes:
- Integrações
- Configurações
- Desempenho

Profundidade esperada:
- média

Permissão típica:
- admin técnico ou leitura avançada

### TI / Segurança
Usa a Canopi para:
- avaliar acesso
- observar integração e governança
- acompanhar riscos técnicos e operacionais

Páginas mais relevantes:
- Integrações
- Configurações

Profundidade esperada:
- baixa a média

Permissão típica:
- admin técnico

## Modelo recomendado de permissão
A Canopi deve combinar três dimensões:
- papel
- escopo
- permissão

### Papel
Exemplo:
- demand gen
- SDR
- RevOps
- liderança
- CS
- mídia
- BI

### Escopo
Exemplo:
- global
- carteira
- segmento
- contas específicas

### Permissão
Exemplo:
- leitura
- comentar
- edição limitada
- edição
- admin

## Regras de produto derivadas
1. o mesmo dado pode aparecer para perfis diferentes com profundidades diferentes
2. a navegação não deve pressupor que todos usam tudo
3. páginas de governança não devem dominar o uso do núcleo operacional
4. a leitura executiva deve ser mais sintética que a operacional
5. perfis técnicos precisam de explicabilidade, não de protagonismo na navegação principal

## Conclusão
A Canopi deve acomodar diferentes perfis sem perder coerência. Isso exige papéis claros, escopos bem definidos e profundidade adequada para cada tipo de usuário.
