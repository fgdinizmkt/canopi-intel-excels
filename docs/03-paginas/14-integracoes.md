# Integrações

## Objetivo da página
Mostrar e governar a saúde das conexões que alimentam a Canopi, deixando claro quais fontes estão ativas, incompletas ou falhando.

## Papel no sistema
A página de Integrações é a camada de sustentação técnica visível do produto.

Ela existe para responder:
- de onde vêm os dados
- o que está conectado
- o que está quebrado
- o que está incompleto

## Perguntas que a página deve responder
- Quais integrações estão ativas?
- Há falhas ou atrasos de sincronização?
- Que dados cada integração alimenta?
- Onde a cobertura do sistema está comprometida?

## O que a página consome
- estado das conexões
- logs básicos
- mapeamentos de fonte
- status de sincronização
- permissões ou credenciais relevantes

## O que a página produz
- visibilidade de saúde técnica
- diagnóstico de cobertura
- apoio à governança operacional
- contexto sobre ausência ou limitação de dados

## Estrutura recomendada da página
- lista de integrações
- status por integração
- última sincronização
- escopo de dados cobertos
- alertas de falha ou ausência

## O que a página não deve fazer
- virar dashboard de negócio
- substituir Configurações
- absorver toda a governança do sistema

## Resumo
A página de Integrações torna visível a infraestrutura de dados da Canopi, ajudando a sustentar confiança e governança.
